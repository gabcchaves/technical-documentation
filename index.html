Introduction

Git is a distributed version control system. It was developed by the Linux community, and in particular by Linux Torvalds, the creator of the Linux Kernel. The Git version control system was intended to help the maintenance of Linux kernel project, aiming some goals, like speed, simplicity, scalability and distribution.

  • Speed
  • Simplicity
  • Scalability
  • Distribution

Install Git

On Linux distributions

It is possible to install Git on Linux distributions either by some package manager or by compiling it from its source code, like on any operating system.

Here are a few examples of how to install Git using package managers:

  • With apt/apt-get, e.g. on Debian based distributions

    sudo apt install git
    sudo apt-get install git

  • With dnf, e.g. on RHEL based distributions

    sudo dnf install git

  • With pacman, e.g. on Arch based distributions

    sudo pacman -S git

  • With emerge, e.g. on Gentoo based distributions

    sudo emerge -q dev-vcs/git

On macOS

To install Git on macOS, issuing the following command on the terminal emulator will prompt the necessary process, if it is not yet installed.

git --version

On Windows

To install Git on Windows, it is possible to download the installer available on https://git-scm.com/download/win.

Start a Git repository

It is possible to create a repository simply emitting the following command on a terminal emulator:

git init repository_name

Alternatively, it is possible to use the current directory or another existing one as a new git repository, by emitting a command like the following:

git init .

Stage files

To stage files on a Git repository means, basically, to prepare them for very next commit, which is an effective version change.

So, before applying any change to the version snapshot, it is necessary to select those new or modified files that are meant to be commited to the snapshot. The following command snippets show how to stage one or multiple files.

git add fileA
git add fileB fileC

The following command snippet shows how to stage all new or modified files in the Git repository.

git add -A

Commit changes

There are two basic ways to commit changes to the version snapshot of the Git repository. The first way allows to pass a version commentary direcly on the command line, as an argument; the second way requires to use a text editor to write such a commentary, usually for more elaborate ones. Here is the first way:

git commit -m "Commentary on the change."

Here is the second way, which will invoke the default text editor, after the command entry:

git commit

Clone Git repository

In order to clone a remote Git repository, that is usually hosted on a service like GitHub or GitLab, there is the following command pattern.

git clone remote_repository_url
